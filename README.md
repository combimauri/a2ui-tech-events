# TechEvents — Generative UI with A2UI + Gemini + Supabase

A small tech-events platform whose screens are **generated by an AI agent** at
runtime using the **[A2UI v0.9](https://a2ui.org/specification/v0.9-a2ui/)**
protocol ("Agent-to-UI"). Instead of hand-coding forms, an admin **describes an
event in plain language** and **Gemini 2.5 Flash** returns a declarative UI
description; the Angular app is a **renderer** that draws it from a fixed,
trusted **component catalogue**.

- **Admin** (any user granted the `admin` role in the database) creates events
  from a natural-language prompt → Gemini drafts an A2UI form → admin
  reviews/edits → submit.
- **Anyone** signs in with a **magic link**, browses events, and **registers**.
- **Data + auth**: Supabase (Postgres with row-level security, magic-link auth).
- **Light & dark themes** with a toggle in the top bar (follows your system setting by
  default, remembers your choice).
- **The agent never ships code** — it can only reference component names that
  exist in the renderer's catalogue. See [docs/a2ui-protocol.md](docs/a2ui-protocol.md).

```
Browser — Angular SPA (A2UI renderer)
  │  supabase-js: magic-link auth + session JWT
  │  functions.invoke('a2ui', { intent, ... })
  ▼
Supabase Edge Function "a2ui" (Deno)
  ├─ intent:"generate" → Gemini → A2UI surface (JSON)
  └─ intent:"action"   → authorize + write DB (RLS) → confirmation surface
  ▼
Supabase Postgres (events, registrations) + Supabase Auth (magic link)
```

Full design notes: [docs/architecture.md](docs/architecture.md) ·
[docs/component-catalogue.md](docs/component-catalogue.md).

## Run it

### 1. Install & start the web app

```bash
cd web
npm install
npm start
```

Open <http://localhost:4200>. The Supabase URL and publishable key are already
configured in `web/src/environments/environment.ts` (both are public by design).

### 2. Set the Gemini API key (required for AI-generated UI)

Get a key from **[Google AI Studio](https://aistudio.google.com/app/apikey)**,
then set it as a secret on the Edge Function (the key is **never** in the
browser):

**Option A — Supabase dashboard**
Project → **Edge Functions** → **Secrets** (or Project Settings → Edge Functions)
→ add:

| Name             | Value                         |
| ---------------- | ----------------------------- |
| `GEMINI_API_KEY` | _your AI Studio key_          |
| `A2UI_MODEL`     | `gemini-2.5-flash` (optional) |

**Option B — Supabase CLI**

```bash
supabase login
supabase link --project-ref ggvlwqosmweggpnfdmte
supabase secrets set GEMINI_API_KEY=YOUR_KEY
```

> **No key yet?** The app still works: the Edge Function falls back to a
> **code-built** form/detail surface so you can click through the whole flow.
> With the key set, the same surfaces are **AI-generated** instead.

### 3. Confirm Supabase Auth settings

In the dashboard → **Authentication** → **URL Configuration**, make sure
`http://localhost:4200` is listed under **Site URL** / **Redirect URLs** so the
magic link returns to your local app. Email sign-in is enabled by default.

## Using the app

1. **Sign in** — enter your email, open the magic link on the same device.
2. **As an admin** (a user with the `admin` role in `user_roles`):
   - Click **Create** → describe the event
     (e.g. _"Free 2-day Angular & AI workshop in Berlin, Oct 14–15, 80 seats"_)
     → **Generate form**.
   - Gemini returns a pre-filled A2UI form. Review/edit, then **Create event**.
3. **As any user**: open an event → **Register** (or **Cancel registration**).

The admin role lives in the database (`user_roles` table) and is enforced
**server-side** by the `is_admin()` RLS helper, which the Edge Function also
consults. The client just calls `is_admin()` to toggle UI affordances; grant
the role with an insert into `user_roles` (see `supabase/migrations`).
